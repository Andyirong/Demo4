# 规范变更历史

本目录记录项目规范的变更历史和提案。

## 变更提案模板

每个规范变更应该包含：

1. **提案** (`proposal.md`) - 说明变更的目的和影响
2. **任务列表** (`tasks.md`) - 实施步骤
3. **设计文档** (`design.md` 可选) - 技术设计细节
4. **更新后的规范** - 新的规范内容

## 变更流程

1. 创建新目录：`YYYY-MM-DD-change-description/`
2. 编写提案文档
3. 团队评审
4. 实施变更
5. 更新主规范文档
6. 归档到此目录

## 示例

```
changes/
├── 2024-12-18-initial-project-structure/
│   ├── proposal.md
│   ├── tasks.md
│   └── design.md
└── 2024-12-20-add-component-guidelines/
    ├── proposal.md
    └── tasks.md
```

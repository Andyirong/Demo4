# Demo4 项目规则

## 分支命名规则

### 格式
```
demo4-XXX
```

### 说明
- **demo4** - 项目标识符，固定前缀
- **XXX** - 三位数字的任务编号，从 001 开始递增

### 示例
- `demo4-001` - 第一个任务分支
- `demo4-002` - 第二个任务分支
- `demo4-003` - 第三个任务分支

### 使用场景
- 所有新功能开发
- Bug 修复
- 项目重构
- 实验性功能

### 注意事项
- 每个 git 分支都必须遵循此命名规则
- 主分支保持为 `main`
- 避免使用 `feature/`、`bugfix/` 等前缀
- 分支名称应简洁明了，便于识别和管理
